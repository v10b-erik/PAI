**Guillermo Rauch:**

Jeg liker strukturen, men la meg utfordre på tech-siden:

**Det som er riktig:**
- AI-pricing som kjerne-feature er smart - det er det som skaper moat
- Freemium → Pro → Marketplace er god trakt
- Tech stack er solid (Next.js + Supabase + Expo)

**Det jeg ville endret:**

1. **Skip appen først.** Start med web-only. React Native tar tid. En PWA som funker på mobil er 80% av verdien med 20% av arbeidet. Ship på 2 uker, ikke 2 måneder.

2. **AI-pricing må være MAGISK fra dag 1.** Hvis den er 60% nøyaktig, dør du. Bruk Claude Vision + få håndverkeren til å justere. Vis "AI forslag: 4.500 kr" med slider. Over tid lærer du fra justeringene.

3. **PDF-tilbud er commodity.** Alle kan lage PDF. Det som er vanskelig er å gjøre hele flyten så smooth at de aldri vil tilbake. Tenk: bilde → pris → send til kunde → kunde godtar → kalender-booking → automatisk påminnelse → jobb ferdig → faktura sendt → betalt. ÉN flyt, null friksjon.

4. **Don't build, integrate.** Fakturering? Bruk Fiken API. Kalender? Google Calendar API. Betaling? Vipps. Du bygger limet, ikke komponentene.

**30-dagers plan er for treg.** Du kan ha MVP live på 7 dager med v0 + Claude + Supabase. Uke 1: live. Uke 2-4: iterer med ekte brukere.
